@1 10 Спадкування

@2 Спадкування
@@ спадкуванн+|нащад+|пред+

@3 Спадкування дозволяє створити новий клас на базі вже існуючого класу.

@3 Нащадок отримує всі поля, методи та властивості предка, і може додати до них власні.
[[Derivation2.png]]

@3 У пам'яті, яку займає об'єкт, поля нащадка слідують за полями предка.

@2 Спадкування та конструктори
@@ base

@3 Клас Point з конструктором.
{{
class Point
{
    public int X {set; get; }
    public int Y {set; get; }

    public Point(int x, int y) {
        X = x;
        Y = y;
    }
}
}}

@3 Конструктор нащадка явно чи неявно викликає конструктор предка для ініціалізації полів, успадкованих від предка.
Для явного виклику батьківського конструктора є спеціальний синтаксис.
{{
class ColoredPoint : Point
{
    public int Color {set; get; }

    Public ColoredPoint(int x, int y, int color): base(x, y)
    {
        Color = color;
    }
}
}}

@2 Перевизначення полів та методів
@@ перевизначенн+

@3 Перевизначення полів можливе, але марне, тому що поля спадкоємця будуть приховувати однойменні поля предка.

@3 Перевизначення методів розглянемо на прикладі.
Нехай у класі Point є метод Info(), який повертає інформацію про стан об'єкта у вигляді рядка.
{{
class Point
{
   public int X {set; get;}
   public int Y {set; get;}

   public string Info() {
      return $"Point: x={X}, y={Y}";
   }
}
}}

@3 Перевизначимо цей метод у класі ColoredPoint.
{{
public string Info() {
    return $"ColoredPoint: x={X}, y={Y}, color={Color}";
}
}}

@2 Додаткові рівні доступу
@@ protected|internal

@3 Спадкування потребує ще одного рівня доступу - protected.
{{
public > protected > private
}}

@3 Щоб закінчити з рівнями доступу членів типу, перерахуємо їх усі:
public
protected
internal - код збірки, що містить тип
protected internal - додає до protected код збірки, що містить тип
private
private protected - додає до private спадкоємців зі збірки, що містить тип (з версії C# 7.2)

@3 Питання: якщо член класу "a" має рівень x, то хто його побачить? 
Замість x підставте всі рівні по черзі.
