@3 Приклад. 0.25 -> ?
{{
0. 25
0 \ 5
1 \ 0
0 \ 0
...
0.25 -> 0.010 ...
}}

@3 {Алгоритм}
Помножити число на 2, отримавши цілу і дробову частини.
Ціле зробити старшим розрядом результату, а над дробовою частиною продовжити роботу.
Коли добуток дорівняється нулю або буде отримано достатньо значущих цифр, зупинитися.

@3 Будь-яке раціональне число (а в комп'ютері числа тільки такі) можна представити періодичної дробом.
Для визначення періоду потрібно аналізувати залишки.
Як тільки якийсь залишок повториться, період обчислення визначиться.
{{
0. 3
0 \ 6
1 \ 2 **
0 \ 4 **
0 \ 8 **
1 \ 6 **
1 \ 2
0 \ 4
0 \ 8
1 \ 6
...
0.3 -> 0.0 (1001) 10011001
}}

@2 Трійкова симетрична система
@@ трійков+ симетричн+

@3 Хоча кількість цифр в позиційній системі дорівнює базі системи, ці цифри не зобов'язані позначати числа 0, 1, 2, ...

@3 В одній з перших ЕОМ, "Сетунь", числа зберігалися в трійковій симетричній системі числення.
Ваги розрядів в цій системі були ступенями трійки, а цифри: 0,1 та i (= -1).
{{
0 - 0
1 - 1
2 - 1i
3 - 10
4 - 11
5 - 1ii
}}

@3 Перевага такої системи в тому, що в ній не потрібна операція віднімання і числа коротші за двійкові (на скільки?).
